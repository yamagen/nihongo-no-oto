# "すり狩衣" を消さないこと

Date: 2026-04-05
Last updated: 2026/04/05-10:56:16

山元啓史

## Summary

「すり狩衣」を gloss するとき、単に `hunting.robe` としてしまうと、「狩衣」であることは示せても、「すり」という加工の要素が消えてしまう。だが原文に「すり」がある以上、それは gloss の中にも残すべきである。ここで重要なのは、日本古典文学を読む外国人研究者のために、原文の要素を減らしすぎないことだ。必要なのは、すべてを説明しきることではなく、要素を見せて、読み手が自分の知識を用いて想像し、理解できる余地を残すことである。

## Body

今日考えたのは、「すり狩衣」をどう gloss するかという問題だった。

最初は、これを一語として扱い、`hunting.robe` とだけ gloss することもできるように思えた。たしかに、それでも狩衣であることはわかる。しかし、そうすると「すり」が消えてしまう。原文には「狩衣」ではなく「すり狩衣」とあるのだから、そこにある加工の要素は書くべきである。

そこで、「すり」と「狩衣」を分けて扱い、

`print hunting.robe`

とする案がよいのではないかと考えた。

ここで大切なのは、`print` が最終的に何を意味するかを、最初から狭く決めすぎないことだ。衣に関する `print` であれば、読み手は自然に、布に模様を施したもの、染めや摺りのような加工を想像するだろう。しかも、これは日本文化に限った特殊な知識とは限らない。衣服に模様や染色や印刷的な加工があることは、多くの文化にまたがって理解可能である。

つまり、日本古典文学を外国人研究者に開くために必要なのは、彼らの知識をいったんすべてキャンセルすることではない。むしろ逆で、彼らがすでにもっている文化的・物質的・文学的知識を使って想像できるように、原文の要素をなるべく消さずに見せることが重要である。

このことは、gloss 全体の作り方にも関わってくる。gloss は、読者に唯一の解釈を押しつけるためのものではない。語の並び、要素の構造、加工や運動の痕跡を示し、それを見た読み手が自分の知識を用いて理解を組み立てられるようにするためのものである。

この観点から見ると、「すり狩衣」を単に `hunting.robe` とするのは情報を減らしすぎている。かといって、`printed and dyed hunting robe` のように詳しくしすぎると、今度は gloss が解釈を先回りしすぎる。したがって、`print hunting.robe` くらいがちょうどよい。要素は残し、説明はしすぎない。その中間が大切なのだと思う。

今回の議論は、小さな gloss の選択のようでいて、実は大きな問題につながっている。日本語を知らない研究者と日本古典文学について議論するとき、必要なのは、すべてを翻訳しきった「完成した意味」ではない。むしろ、どのような要素がそこにあるのかを見せることであり、その上で逐語訳や自然訳を参考にしながら、読者自身が理解に参加できるようにすることだ。

「すり狩衣」の `すり` を書くべきだという今日の結論は、そのまま、この研究全体の方針にもつながっている。

## Notes

今回の gloss の候補は、次のようになる。

`すり 狩衣 の たもと に 書き つけ ける`
`print hunting.robe GEN sleeve LOC write attach PST`

このようにすると、「すり」が消えず、しかも読み手の想像の余地も残る。

とくに

`書き / つけ / ける`
`write / attach / PST`

という並びが、そのまま見えるのがよいです。`書きつく` を最初から一語の自然訳にしてしまわず、

- 書く
- 付ける
- 過去

という連鎖として出しているので、先生の方式によく合っています。

この箇所では、`attach.ADV` がとても効いていますね。さきほどの `もとつきにける` の `attach` とも呼応しますし、「付く」「付ける」の芯を保ったまま、文脈で読み手が理解できるようになっています。こういう統一は大きいです。

`たもと` を `sleeve` としたのも、gloss-line では十分です。より細かく言えば「袂」なので単純な sleeve より広がりはありますが、この方式では短く置くほうが大事なので、これでよいと思います。

つまり、全体としてはこのままで十分通ります。

もし細かく見るなら、一点だけ、`phrase` の `すり狩衣` をどう扱うかは今後の統一ポイントになるかもしれません。たとえば、

- `すり狩衣` を一語として扱う
- `すり / 狩衣` に分ける

のどちらかです。ただ、この箇所では一語として見せたほうが自然ですし、`words` でも `hunting.robe` と一語で受けているので、現状はこのままでよいです。

> そうですね。"すり"は、"摺り"なんですね。これにどういうglossをつけますか？

はい。「すり」はここでは「摺り」ですね。

`すり狩衣` を分けるなら、この `すり` は、布に模様を摺りつけたことを表すので、gloss としてはまず

`print`

がいちばん扱いやすいと思います。

つまり、

```json
{ "word": "すり", "gloss": "print" },
{ "word": "狩衣", "gloss": "hunting.robe" }
```

です。

これなら `書き / つけ` を `write / attach` としたのと同じく、あまり説明的になりすぎず、芯だけを残せます。英語として自然訳に寄せすぎると `patterned` や `printed` も候補になりますが、先生の方式では、gloss-line は語の対応を見せるものですから、まずは動作の芯を残して `print` がよいです。

ただし、`すり狩衣` を一語として扱うなら、今のまま `hunting.robe` でも十分です。分ける場合にだけ `すり = print` を立てればよいです。

私なら、分けるなら `print` にします。`rub` だと工程寄りすぎ、`patterned` だと結果寄りすぎるからです。`print` がいちばん中立で使いやすいです。

この方式、本当に安定してきましたね。

> 御提案のようにhunting.robe だけだったら、狩衣でいいわけです。しかし、"すり"がついていますから、書くべきですよ。その上で読者はprintの種類を考えると思いますが、衣に関するprint ですから、それは染物をおそらく想像します。これは日本文化に限られたことではないと思います。そういう想像転移を文化を越えてすることが重要だと思います。外国人研究者の知識が日本古典文学を取り扱うために、すべての研究者の知識をキャンセルする必要はないですよね。

その通りです。まさにそこが重要ですね。

`狩衣` だけで済ませてしまうと、「何の狩衣か」は見えても、「どういう加工が施されたものか」が消えてしまいます。`すり` がある以上、それは書くべきです。しかも、それを書いたからといって、読み手に過剰な説明を押しつけるわけではありません。むしろ、読み手が自分の側の知識を使って想像するための足場になります。

おっしゃるように、衣に関する `print` であれば、多くの読者は自然に「布に模様を施したもの」「染めや摺りのような加工」を思い浮かべるはずです。そこを最初から全部説明語に変えてしまうより、`print` とだけ置いておくほうが、文化を越えた想像の余地が残ります。これは非常に大事です。

つまり、ここで必要なのは「日本文化の細部を知らない読者のために全部を平板化すること」ではなく、

- 原文にある要素は消さない
- ただし過剰に説明しない
- 読者の既存の知識による想像を許す

ということですね。

これは先生のフォーマット全体の考え方にも一致しています。逐語訳も gloss も、読む人の知識を全部キャンセルして「唯一の理解」に閉じ込めるためのものではなく、むしろ

**原文の要素を見せて、そこから読者が自分の知識で理解を組み立てられるようにする**

ためのものです。

その意味で、`すり` を落とさずに `print` と書くのはとてもよいと思います。`dyed` だと解釈を先回りしすぎますし、`patterned` だと結果だけになって工程感が消えます。`print` なら、加工があることだけを示し、具体的な種類は読者の想像に委ねられます。

がよいと思います。
